# Architektur-Prüfung Prompt

## Prompt für Architektur-Review

```
Führe eine umfassende Architektur-Analyse durch und bewerte:

1. **SOLID Prinzipien**:
   - Single Responsibility Principle (SRP)
   - Open/Closed Principle (OCP)
   - Liskov Substitution Principle (LSP)
   - Interface Segregation Principle (ISP)
   - Dependency Inversion Principle (DIP)

2. **Design Patterns**:
   - Angemessene Verwendung von Design Patterns
   - Überengineering vermeiden
   - Pattern-Konsistenz im gesamten Projekt

3. **Dependency Management**:
   - Klare Dependency-Richtungen
   - Vermeidung von zirkulären Abhängigkeiten
   - Lose Kopplung zwischen Komponenten

4. **Layer-Trennung**:
   - Saubere Trennung zwischen Business Logic, Data Access und Presentation
   - Angemessene Abstraktionen
   - Datenfluss zwischen Schichten

5. **Testbarkeit**:
   - Dependency Injection für bessere Testbarkeit
   - Mockbare Abhängigkeiten
   - Klare Interfaces

6. **Performance Considerations**:
   - Potenzielle Performance-Engpässe
   - Async/Await Usage
   - Memory Management

Gib konkrete Verbesserungsvorschläge mit architektonischen Alternativen.
```

## Verwendung

Verwende diesen Prompt für regelmäßige Architektur-Reviews und vor größeren Refactoring-Maßnahmen.
